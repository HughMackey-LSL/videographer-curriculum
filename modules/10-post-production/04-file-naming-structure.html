File Naming and Folder Structure

~8 min read4 objectives

Learning Objectives

  • Construct a file name that communicates date, project, camera roll, and clip number.
  • Describe a recommended folder hierarchy for a video project handoff.
  • Explain why the RAW folder should never be modified after delivery.
  • Identify what belongs in a post-shoot notes file.

The Folder Opens and the Editor Forms a First Impression

The first thing an editor does when they receive your media is open the project folder. In that first few seconds, they learn something about you as a professional. A well-organized folder tells them: this shooter knows how post-production works, this media is safe to use, and I can start editing quickly. A disorganized folder tells them something different.

File organization is not glamorous work. It does not appear on screen. The audience never knows whether the clips were labeled systematically or haphazardly. But the editor knows — and the time wasted on disorganized media is real time, measured in hours, that could have been spent on the edit itself.

The Problem with Camera-Generated File Names

Cameras generate file names automatically. A Sony camera might produce C0001.MP4, C0002.MP4. A Canon might produce MVI_3847.MOV. A DJI drone might produce DJI_0042.MP4. These names communicate almost nothing about the content. They convey no information about the project, the shoot date, which camera the clip came from, or what the clip contains.

When an editor receives a folder containing hundreds of clips named by camera default, they have no way to understand the material without opening each clip individually. In a multi-camera shoot where two cameras both used default naming and their clip numbers overlap, the editor cannot tell which clips came from which camera without checking the metadata of each file.

Renaming files before delivery is a professional responsibility. It takes additional time but eliminates confusion for everyone downstream — including yourself when you look at the project six months later.

A Good File Naming Convention

A well-structured file name answers these questions at a glance: When was this shot? What project is it for? Which camera recorded it? What is the clip number in sequence?

A practical convention that covers all of these:

YYYYMMDD_ProjectName_CamLetter_RollNumber_ClipNumber.ext

For example:

20240315_StaffInterview_A_R01_0023.MP4
20240315_StaffInterview_B_R01_0008.MP4

Breaking this down:

  • 20240315 — The shoot date in YYYY-MM-DD format. This format sorts chronologically when files are sorted alphabetically, which is a significant practical advantage.
  • StaffInterview — The project name or short descriptor. Keep it concise and avoid spaces (use underscores or CamelCase instead, as spaces cause problems in some software and command-line environments).
  • A — Camera identifier. Camera A is the primary camera; Camera B is the secondary. This distinguishes clips from different cameras when clip numbers overlap.
  • R01 — Roll number. Cameras reset their clip counter when a card is formatted. A roll number distinguishes Card 1 from Card 2 from Card 3 within a project.
  • 0023 — The clip number from the camera, zero-padded to four digits so files sort correctly.
Tip

Always confirm the naming convention with the production team or editor before the shoot. Some organizations have established naming standards that differ from any generic convention. Using their standard ensures your media integrates seamlessly with their existing project structure.

Recommended Folder Structure

Folder structure should separate raw media, working files, and project outputs into clearly labeled containers. A functional hierarchy for a typical video project:

ProjectName/
  YYYYMMDD/
    RAW/
      Camera_A/
      Camera_B/
      Audio/
    Exports/
    Notes/

Breaking this down by purpose:

RAW/

The RAW folder contains the original camera files and audio recordings, exactly as they came off the cards. This folder is the archive. It should never be modified, rearranged, or have files deleted from it once the media has been verified and backed up. Editors work from copies, not from the RAW folder directly. The RAW folder is the source of truth, and its integrity is paramount.

Camera_A/ and Camera_B/

Separate subfolders for each camera's media prevent clip number collisions and make it immediately clear which clips came from which camera. If Camera A and Camera B both have a clip numbered 0042, they coexist without conflict in their respective subfolders.

Audio/

Audio recorder files belong here: WAV files from a field recorder, boom operator recordings, or lavelier backup recordings. Keeping them separate from video files but within the RAW hierarchy makes them easy to find during the sync pass.

Exports/

Rendered output files go here: rough cut exports for client review, final delivery files, compressed social media versions. Keeping exports separate from raw media prevents any possibility of confusing a re-encoded file with an original.

Notes/

Documentation about the shoot: the shot list, take notes, any problems encountered, anything the editor should know. A plain text file or PDF. More on this below.

Important

Never format a card until you have verified the backup. The workflow is: copy to primary drive → verify copy → copy to backup drive → verify backup → format card. Skipping verification or formatting before completing both copies is how footage is permanently lost. A card format is irreversible.

The Notes File

A notes file is a simple document included with the media handoff that gives the editor context they cannot get from watching the footage alone. It does not need to be elaborate — a plain text file is sufficient — but it should include:

  • Shot list with clip references — Which clips correspond to which shots. "Staff interview B-roll: Camera A clips 0031–0048."
  • Take notes — Which takes are the best. "R02_0012 is the preferred take of the opening statement. R02_0010 and R02_0011 had stumbles."
  • Technical issues — Any known problems the editor should be aware of. "Audio on Camera B clips R01_0003 through R01_0007 has slight RF noise — use Camera A reference track for sync only."
  • Pickup needs — Anything that could not be captured and may require a second unit or pickup shoot. "No B-roll of the loading dock — mentioned in interview segment 3."
  • Location context — Anything that explains choices made on set that might otherwise look like errors. "Interview was moved to the boardroom due to HVAC noise in the original location."
Note

A notes file written immediately after the shoot is far more accurate than one written two days later. Capture take notes during the shoot, while the details are fresh. Even brief notes ("take 3 is best") are more useful than no notes at all.

Delivering to Shared Storage

On larger productions, media may be delivered to a shared network drive or server rather than a personal drive. When this is the case, several additional checks apply:

  • Confirm the destination — Get the exact folder path in writing from the producer or post coordinator. Do not guess or approximate.
  • Confirm the naming convention — Shared storage environments often have established naming standards. Match them exactly, even if they differ from your personal preference.
  • Verify the transfer completed — Do not assume a file copy to a network drive succeeded. Compare file counts and total sizes between source and destination.
  • Confirm backup exists — On shared storage, ask who is responsible for backup and confirm it has been performed before formatting cards.

If Footage Is Misnamed After Delivery

If you realize footage was misnamed or misorganized after handing off to the editor, notify them immediately. Do not silently rename or reorganize files in a project folder the editor has already begun working with — this breaks links in the edit project and can cause significant problems. Communicate the issue, describe what needs to change, and coordinate the fix with them so they can update their project accordingly.

Key Takeaways

  • Camera-generated file names communicate nothing useful; rename files before delivery using a convention that includes date, project, camera, roll, and clip number.
  • A logical folder hierarchy (RAW → Camera_A, Camera_B, Audio; Exports; Notes) keeps raw media, working files, and documentation cleanly separated.
  • The RAW folder is the archive — never modify, rearrange, or delete from it after delivery.
  • A notes file gives the editor context they cannot get from the footage alone: preferred takes, technical issues, pickup needs, and location context.
  • Never format a card before verifying that the backup copy is complete and intact.
